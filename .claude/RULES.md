# Claude 작업 기본 수칙

이 파일은 Claude가 이 프로젝트에서 작업할 때 반드시 따라야 할 규칙입니다.

---

## 1. 데이터 검증 필수

### 절대 하지 말 것:
- 로또 번호, 날짜, 통계 등 **실제 데이터를 추측하거나 만들어내지 말 것**
- 기존 코드에 있는 데이터가 "진짜"라고 가정하지 말 것
- "그럴듯해 보이는" 데이터를 검증 없이 추가하지 말 것

### 반드시 할 것:
- 실제 데이터가 필요하면 **WebSearch로 공식 출처 확인** 후 추가
- 데이터 출처를 코드 주석에 명시 (예: `// 출처: 동행복권 공식`)
- 검증할 수 없으면 사용자에게 **솔직하게 "확인 필요"** 라고 말하기

---

## 2. 불확실할 때 솔직하게 말하기

### 절대 하지 말 것:
- 모르는 것을 아는 척하지 말 것
- 추측을 사실처럼 말하지 말 것
- 에러가 나면 숨기지 말 것

### 반드시 할 것:
- "확실하지 않습니다" / "검증이 필요합니다" 라고 말하기
- 추측일 경우 "추측입니다" 라고 명시
- 문제가 생기면 즉시 사용자에게 알리기

---

## 3. 외부 데이터 다루기

### 로또/복권 데이터:
- 동행복권 공식 사이트 또는 뉴스 기사로 검증
- 회차 번호, 날짜, 당첨번호, 보너스 번호 모두 확인
- API 사용 시 CORS 제한 고려

### 날짜/시간:
- 오늘 날짜: 시스템에서 확인
- 과거 이벤트 날짜: 웹 검색으로 확인

### 가격/통계:
- 공식 출처에서 확인
- 출처 URL 첨부

---

## 4. 코드 작성 시

### 테스트 데이터 vs 실제 데이터:
- 테스트용 더미 데이터는 `// TEST DATA - 실제 데이터 아님` 주석 필수
- 실제 데이터는 `// 출처: [URL]` 또는 `// 검증일: YYYY-MM-DD` 명시

### 하드코딩된 값:
- 왜 이 값인지 주석으로 설명
- 마법 숫자(magic number) 사용 금지

---

## 5. 사용자에게 보고할 때

### 작업 완료 보고:
- 무엇을 했는지 구체적으로
- 검증 여부 명시
- 문제점이 있으면 함께 보고

### 에러/문제 발생 시:
- 숨기지 말고 바로 보고
- 원인과 해결 방안 제시
- "모르겠다"면 솔직히 말하기

## 7. 이 프로젝트 특이사항

### 로또 분석기 프로젝트:
- 동행복권 API: `https://www.dhlottery.co.kr/common.do?method=getLottoNumber&drwNo={회차}`
- CORS 제한으로 브라우저/앱에서 직접 호출 불가
- 정적 데이터 사용 시 반드시 공식 출처에서 검증

### 플랫폼:
- 웹: Blazor WebAssembly (`LottoAnalyzer.Web`)
- 앱: React Native Expo (`LottoApp`)
- 공용: .NET Core (`LottoAnalyzer.Core`)
